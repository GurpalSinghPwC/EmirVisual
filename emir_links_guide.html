Guide

This mini-site helps stakeholders understand dependencies between EMIR fields (mains) and the cleaned items they reference (links). The data comes from the Excel table Table1_1 in your workbook.

Pages

  • Network — interactive graph of Field ↔ Cleaned links. Big orange nodes are Fields; grey dots are linked items. Size reflects number of connections.
  • Cousins — linked items that are shared by two or more Fields. The “Cousin strength” is how many Fields share that item.
  • Orphans — nodes that have no connections (rare in practice; useful for QA).

Network: how to use

  • Search: type a few letters in the right panel to filter the list; click a result to zoom.
  • Highlight: emphasises matches and dims others. Clear resets.
  • Color mode: Role (Field vs Linked), Degree (more connections → deeper colour), Communities (algorithmic clusters).
  • Hover a node: emphasises its neighbours and edges (“edge emphasis”).
  • Snapshot: downloads a PNG of the current view (good for slides).
  • Top hubs: fields with the most links; click to focus.

Data model

  • Main column: Field (only this creates orange Field nodes)
  • Linked column: Cleaned (creates grey linked nodes; multiple values allowed and can be separated by commas/semicolons)
  • Canonicalisation: all labels are converted to Title Case (e.g., “asset class” → “Asset Class”) to merge trivial duplicates.

Notes & Tips

  • If a value should not be merged by Title Case (e.g., IDs), keep the canonical value consistent in the source sheet.
  • For best readability, keep linked items concise and avoid long compound phrases.
